# 修正問題

````md magic-move
```jsx {all|1,4-9|21-28}
export default function Page() {
  const [data, setData] = useState(initialData);

  /* 👇 原本在 DataInputTab 裡面的狀態移到父組件 <Page/> */
  const [newData, setNewData] = useState({
    date: new Date().toISOString().split('T')[0],
    visitors: 0,
    active: 0,
  });

  return (
    <>
      <FilterForm />

      <Tabs defaultValue="add" className="mt-4 w-full">
        <TabsList className="grid w-full grid-cols-2">
          <TabsTrigger value="add">新增資料</TabsTrigger>
          <TabsTrigger value="other">用戶設置</TabsTrigger>
        </TabsList>

        {/* 👇 將 newData 和 setNewData 往下傳 */}
        <TabsContent value="add">
          <DataInputTab
            setData={setData}
            newData={newData}
            setNewData={setNewData}
          />
        </TabsContent>

        <TabsContent value="other">
          <SettingsTab />
        </TabsContent>
      </Tabs>

      <HeavyCharts data={data} />
    </>
  );
}
```
````

# 什麼是 `<Activity />`

`<Activity />` 可以讓我們將狀態保留在各自的組件中，並不會因為組件被 unmount 而導致狀態丟失。

````md magic-move
```jsx {all|3,5,7,9}
import { Activity } from 'react';

<Activity mode="visible | hidden">
  <SomeComponent />
</Activity>

<Activity mode={isActive ? 'visible' : 'hidden'}>
  <SomeComponent />
</Activity>
```
````

<v-click>

**當 `mode="hidden"` 時，組件不會顯示但 state 與 DOM 會被保留下來。**

</v-click>

<!--
`<Activity />` 可以讓我們將狀態保留在各自的組件中，並不會因為組件被 unmount 而導致狀態丟失。

使用方法很簡單，我們可以傳入 mode 來控制組件是否顯示：

visible 就是顯示組件，hidden 就是隱藏組件。

而當 mode="hidden" 時，雖然會隱藏了組件，但是 state 與 DOM 會被保存下來。不會因為組件被隱藏而導致狀態丟失。
-->

# 用 `<Activity />` 重寫 Tabs

```jsx{all|8-31}
export function PageTabs({ setData }) {
  const [activeTab, setActiveTab] = useState('add');

  return (
    <Tabs>
      <TabsList className="grid w-full grid-cols-4">...</TabsList>

      <Activity mode={activeTab === 'add' ? 'visible' : 'hidden'}>
        <TabsContent value="add">
          <DataInputTab setData={setData} />
        </TabsContent>
      </Activity>

      <Activity mode={activeTab === 'settings' ? 'visible' : 'hidden'}>
        <TabsContent value="settings">
          <SettingsTab />
        </TabsContent>
      </Activity>

      <Activity mode={activeTab === 'filter' ? 'visible' : 'hidden'}>
        <TabsContent value="filter">
          <ReportFilterTab />
        </TabsContent>
      </Activity>

      <Activity mode={activeTab === 'view' ? 'visible' : 'hidden'}>
        <TabsContent value="view">
          <ViewOptionsTab />
        </TabsContent>
      </Activity>
    </Tabs>
  );
}
```

# 為什麼 `<Activity />` 可以保留狀態？

其實 `<Activity />` 的原理不困難，他是利用 `display: none` 來控制是否顯示組件。

<br>

<v-click>

雖然組件本身沒有被 unMount 但 Effect 仍然會被清空，所以當 `<Activity mode=”visible” />` 時

**仍然會執行 Effect 裡面的邏輯。**

</v-click>

<br>

<v-click>

不過也是因為組件本身沒有被 unmount，所以當父組件 Re-render 時

`<Activity />` 內部的組件也會被影響，不管他現在是不是顯示的狀態。

</v-click>
